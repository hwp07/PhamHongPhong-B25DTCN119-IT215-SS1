PHẦN 1: KHÁI NIỆM CƠ BẢN
1. Backend là gì?
Backend là những phần hậu trường, những phần mà người dùng không thể nhìn thấy trực tiếp và thao tác với chúng (máy chủ, cơ sở dữ liệu, …..)

2. Client – Server là gì?
Là mô hình quan hệ thể hiện mối tương quan giữa backend(server) và frontend(client), client thao tác ở frontend và server nhận - xử lý thao tác đó

3. API là gì?
API là 1 bên trung gian, cầu nối giữa FE và BE,  cho phép BE và FE giao tiếp với nhau 

4. JSON là gì?
JSON là 1 file định dạng chứa các cặp key - value (“ten”: “Phong”)

PHẦN 2: KIẾN THỨC WEB CƠ BẢN
1. HTTP là gì?
Là phương thức trao đổi dữ liệu giữa client và server

2. GET / POST / PUT / DELETE dùng để làm gì?
GET: lấy dữ liệu từ server
POST: tạo dữ liệu
PUT: cập nhật dữ liệu
DELETE: xóa dữ liệu

PHẦN 3: FastAPI
1. FastAPI là gì?
FastAPI là một framework của Python dùng để xây dựng API nhanh và dễ dàng

2. FastAPI dùng để làm gì?
FastAPI được dùng để xây dựng BE cho ứng dụng, tạo các API để trao đổi dữ liệu giữa FE và BE

3. Theo các em FastAPI khác gì so với web framework truyền thống?
FastAPI chuyên làm API, nhanh hơn và tự động hóa nhiều thứ hơn
framework truyền thống => làm tất cả mọi thứ từ FE đến BE
